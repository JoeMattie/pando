Usage (Web)

caution

This is experimental documentation for Pluralsight Design icons package.

The Icons package provides inline SVG icons as ready-to-use components for popular frameworks as well as SVG files and data URIs.

View source

Import​

We export React components by default, which can be imported by name from the package root.

React​

import { MenuIcon } from '@pluralsight/icons'

Svelte​

import { MenuIcon } from '@pluralsight/icons/svelte'

SVG​

import menuSvg from '@pluralsight/icons/svg/navigation/menu.svg'

Typescript and pathed imports​

To use pathed imports (those other than @pluralsight/icons) with Typescript you will need to set moduleResolution to nodenext

tsconfig.json
{
  "compilerOptions": {
    "moduleResolution": "nodenext"
  }
}

If you are unable to upgrade or use these settings, you can reference the build files directly.

  • React
  • Svelte
  • SVG
import MenuIcon from '@pluralsight/icons/build/react/navigation/MenuIcon.js'
import MenuIcon from '@pluralsight/icons/build/svelte/navigation/MenuIcon.svelte'
import menuSvg from '@pluralsight/icons/build/svg/navigation/menu.svg'
tip

If your bundler does not support tree shaking, please see our notes on minimizing bundle size

Usage​

Basic example​

Icons are provided as either SVG or component exports based on your choosing.

<MenuIcon {...getIconProps()} />
tip

The components provided from this library only contain an svg element to allow for maximum flexibility. Additional attributes, such as sizing, will be provided by the headless-styles package, which can also be applied to custom icons.

Data URI​

If you need to embed an icon in an img tag or using for a CSS property value, you can use the getIconString method.

Example using an img src value
import { getIconString } from '@pluralsight/icons/string'

function MenuIcon(props) {
  return (
    <img alt="menu icon" src={getIconString('MenuIcon', props.color)} />
  )
}

Here is an example using getIconString as a CSS value.

Example using a CSS property value
import { getIconString } from '@pluralsight/icons/string'

function MenuIcon(props) {
  return (
    <span
      aria-label="menu icon"
      style={{
        backgroundImage: `url("${getIconString('MenuIcon', props.color)}")`,
        display: 'inline-block',
        height: '1rem',
        width: '1rem',
      }}
    ></span>
  )
}

Icon colors​

By default the icons are monochromatic and will inherit the color of their parent. This ensures the icon color matches its context by default and facilitates theming while still allowing for control over the icon's color directly.

<button
  style={{
    backgroundColor: 'var(--ps-background-weak)',
    borderRadius: '6px',
    color: 'var(--ps-text)',
  }}
>
  <PlaceholderIcon {...getIconProps()} />
  Icon matches font color
</button>

Icon sizes​

Icons come in three size variants: s, m (default), l.

<StarIcon {...getIconProps({ size: 's' })} />
<StarIcon {...getIconProps({ size: 'm' })} />
<StarIcon {...getIconProps({ size: 'l' })} />

Icon size guide​

We have defined four standard sizes for icons. This sizing should be applied using the headless-styles package.

Example interface - does not exist in types
interface IconSizeGuide {
  s: {
    height: '1rem'
    width: '1rem'
  }
  m: {
    height: '1.25rem'
    width: '1.25rem'
  }
  l: {
    height: '1.5rem'
    width: '1.5rem'
  }
}

Badge

caution

This is unreleased documentation for headless-styles package.

The Badge is used to highlight a category for quick recognition.

View source

Import​

import { getBadgeProps } from '@pluralsight/headless-styles'

Basic Badge​

The Badge comes with three variants: strong (default), medium, and weak.

DefaultStrongMediumWeak
<span {...getBadgeProps()}>Default</span>
<span {...getBadgeProps({ kind: 'strong' })}>Strong</span>
<span {...getBadgeProps({ kind: 'medium' })}>Medium</span>
<span {...getBadgeProps({ kind: 'weak' })}>Weak</span>

CSS API​

getBadgeProps​

getBadgeProps(options: BadgeOptions): BadgeProps

The Badge prop-getter takes in BadgeOptions returns an Object that contains all the a11y attributes and styles for you to use in any fashion you like - or easily extend/overwrite when needed.

Return value example
{
  className: `ps-badge ${...internal-style-modules}`
}

BadgeProps​

interface BadgeProps {
  className: string
}

CSS in JS API​

getJSBadgeProps​

getJSBadgeProps(options: BadgeOptions): JSBadgeProps

If you prefer to use CSS-in-JS, use the getJSBadgeProps function. This will return a stringified version of the styles via the cssProps property along with a styles property that matches the common emotion framework syntax.

Return value example
{
  cssProps: `
    color: #fff;
    background-color: #000;
    ...
  `,
  styles: {
    color: '#fff',
    backgroundColor: '#000'
    '&:hover': {
      ...
    }
  }
}

JSBadgeProps​

interface JSBadgeProps {
  cssProps: string
  styles: Record<CSSProps, string>
}

Extending​

There are times you may need to make a slight adjustment to the Badge which is easy since we are giving you an Object. You can easily extend the Badge in any way that you like.

Here is an example of using styled-components to extend a Badge color.

page/Login/components/SubmitButton.jsx
import styled from 'styled-components'
import { getJSBadgeProps } from '@pluralsight/headless-styles'

const psBadgeProps = getJSBadgeProps({ kind: 'medium' })

const SuccessBadge = styled.span`
  ${psBadgeProps.cssProps}
  background-color: 'green'
`

export default SuccessBadge

Using with other frameworks​

headless-styles was built with the mindset that any tech stack should work with our helpers. However, when making the decision to support multiple UI frameworks, styles can get out of hand very quickly. In order to scale styles without creating negative impacts to other platforms, we have included the optional tech option to BadgeOptions. This small inclusion allows us to ship specific styles per platform you choose to use without conflicting with others. 💥

tip

When combining headless-styles with other frameworks, please follow the guides in the example codesandboxes for the best experience. Not all frameworks require special CSS support.

BadgeOptions​

interface BadgeOptions {
  kind?: 'weak' | 'medium' | 'strong'
  tech?: 'svelte'
}

Badge color guide​

Badge colors are relative to the background style and have a direct correlation from the kind to the design-tokens package. Here is an interface that gives the color guide.

Example interface - does not exist in types
interface BadgeColorGuide {
  weak: 'var(--ps-neutral-border-weak)'
  medium: 'var(--ps-neutral-border-weak)'
  strong: 'var(--ps-neutral-background-hover)'
}

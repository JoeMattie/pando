Intro

caution

This is unreleased documentation for headless-styles package.

Install Pluralsight's headless UI framework that works with any library or tech stack.

@pluralsight/headless-styles

npm​

To use Headless Styles install the package with your preferred manager.

  • npm
  • Yarn
npm install @pluralsight/headless-styles@alpha
yarn add @pluralsight/headless-styles@alpha
note

Some component examples use the Icons package. Not all components require icons, so we do not include it in the above install recommendation.

Headless design​

The Headless Styles library is a component-less tool that allows you to obtain all the styles needed for components without dictating a tech-stack for your team to use. This means that you can theoretically use any combination of technology (i.e. React, Svelte, MUI, etc.) and the library should still work successfully for your team.

Having said that, it will always be impossible to cover every single use case so if your team comes across an instance where the styles are not matching the doc examples, please file a bug report so we can make sure that use case is covered.

Examples​

For each component, we provide mutliple examples using the most popular libraries. Out of the box, we provide examples for:

  • React
  • styled-components
  • Chakra UI
  • MUI
  • Svelte

If there is an additional library you would like to see provided in the examples, please start a poll and if it receives enough support, we will be happy to add it in. 😄

Dependencies​

caution

Headless Styles requires the use of our fonts and normalize setup. Please see the installation guide for more information.

CSS & CSS-in-JS​

Each of our Headless Styles components will deliver APIs for both CSS and CSS-in-JS use cases. Both APIs also come with some requirements for your project that you should be aware of for the most success.

Themeing​

Internally, we use the Design Tokens to control all the colors presented. Our suite comes with multiple themes that work natively for CSS environments. This means that if your team uses CSS-in-JS, you will need to include a provider to use theming.

CSS Modules​

For the CSS APIs, we use CSS modules which allow us to deliver the best experience regarding performance and scalability. Most tools come with CSS module support baked in (i.e. CRA, NextJS, Vite).

If you are using a bundler with a custom config via Webpack (or any other), you will need to make sure you add a loader which should be available for any tool you are using.

CSS-in-JS​

Our CSS-in-JS APIs source a generated style Object we create internally which delivers the "dark" theme by default. If you want to use a custom theme, please see our theme customization in the Tokens docs.

The easiest solution is to just extend this library with whatever tool you prefer to use (i.e. React, styled-components, etc.) by following each components "extending" section example.

note

If you would like the Design System to provide a ThemeProvider solution, please contribute to our discussion poll.

Chakra UI​

We prefer not to cater to frameworks (which is the reason we designed this package the way we have). However, Chakra does provide a way for us to deliver style objects to overwrite their UI while not having to add it as a dependency to ours. 💥

Because of this, all of our "components" will return Chakra style objects instead of using a prop-getter function.

If you use Chakra, please be mindful of this and all the examples we provide (which should work in your favor).

How to use with Chakra
import { theme } from '@chakra-ui/react'
import { chakraPSTheme } from '@pluralsight/headless-styles'

const theme = extendTheme(chakraPSTheme)

Once you add the theme to your ChakraProvider you can use Chakra-UI components and the ones our Web UI Kit supports will automatically be styled for you!

note

Some components (like the Skeleton) will require some additional configuration setup to the theme config (see the Chakra example) due to the nature of Chakra being tightly coupled with Emotion.

Recommendation​

Our recommendation for the best user experience is to stay 3rd-party free and just stick with your Native library (i.e. React, Svelte, etc.) which will yield the best experience of our library.

Having said that, we know people are very passionate about UI libraries and believe they must be used in a project. Whatever your decision is, Headless-Styles aims to work with as many libraries as possible - there just may be a little extra work you will have to put in, whether updating types or customizing for CSS animations.

This is due to our decision to stay as light-weight as possible and not require any additional requirements for you to use our project (i.e. you will not have to install any other package to use any of our libraries).

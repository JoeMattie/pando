Avatar

caution

This is pre-release documention for the experimental version of the headless-styles package.

The Avatar is used to represent a user and is designed to display a profile image, initials, or fallback icon.

View source

Import​

import { getAvatarProps } from '@pluralsight/headless-styles'

Usage​

Avatar Kinds (fallback only)​

The getAvatarProps helper has two kind's to change the visual color of the avatar fallback. You can use neutral (default) and strong.

note

When using Avatars, make sure there is a purpose behind the fallback colors you are choosing. Follow our guide to learn how to display avatars for the best user experience.

NASA
const { avatar } = getAvatarProps({
  ariaLabel: 'Pluralsight avatar'
  kind: 'strong',
})

<span {...avatar}>PS</span>

Avatar Sizes​

To change the size of an Avatar, use the size option. The options are: xs, s, m (default), l, and xl.

xssmlxl
const { avatar } = getAvatarProps({
  ariaLabel: 'extra large avatar',
  size: 'xl'
})

<span {...avatar}>xl</span>
tip

We use the standard clothing size shorthand values for all size related options in any library we provide (i.e. xs-xxl).

Avatar Content​

The DS Avatars have three different states to display to the user. Each state has a different purpose for usage and should follow the guides below.

Image (preferred)​

A profile image should be used for all Avatars no matter the context (i.e. person, platform, etc.). This will give the best visual experience for the user and should be used over all other options.

const { avatar, image } = getAvatarProps({ ariaLabel:  name })

<span {...avatar}>
  <img {...image} src={src} alt={initials} />
</span>

Initials (fallback)​

In the absence of an image, the avatar's initials (up to 2 characters) should be displayed. This is not ideal but allows the user to still quickly put the dots together of who the avatar is meant to represent.

const { avatar } = getAvatarProps({ ariaLabel:  name })

<span {...avatar}>{initials}</span>

Icon (last resort)​

If neither the image nor initials are available, display an icon using the iconOptions from the AvatarProps Object to represent a person.

danger

Although this option exists, it is not recommended to be used as it provides a vague user experience. If you have to use this option, you need to update your API to require a name for each user.

const { avatar, iconOptions } = getAvatarProps({ ariaLabel:  name })

<span {...avatar}>
  <PersonIcon {...getIconProps(iconOptions)} />
</span>

Options​

AvatarOptions​

This interface is available as an import for use.

export interface AvatarOptions {
  ariaLabel?: string
  kind?: 'neutral' | 'strong'
  size?: 'xs' | 's' | 'm' | 'l' | 'xl'
  tech?: 'svelte' | ''
}
Import example
import type { AvatarOptions } from '@pluralsight/headless-styles/types'

Props​

AvatarProps​

interface AvatarProps {
  avatar: {
    'aria-label': string
    className: string
  }
  image: {
    'aria-hidden': true
    className: string
  }
  iconOptions: {
    ariaHidden: true
    customSize: string
    tech: string
  }
}

JSAvatarProps​

interface JSAvatarProps {
  avatar: {
    a11yProps: {
      'aria-label': string
    }
    cssProps: TemplateLiteralStringArray | string
    styles: Record<CSSProps, string>
  }
  image: {
    a11yProps: {
      'aria-hidden': true
    }
    cssProps: TemplateLiteralStringArray | string
    styles: Record<CSSProps, string>
  }
  iconOptions: {
    ariaHidden: true
    customSize: string
    tech: string
  }
}
